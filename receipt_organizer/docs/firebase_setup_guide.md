# Real-Time Shared Database Implementation Plan

## Goal Description
현재 각자의 스마트폰에서 독립적으로 동작하는 앱을 **"모든 사람이 실시간으로 똑같은 표(데이터베이스)를 공유하는 시스템"**으로 업그레이드합니다. 직원이 영수증을 올리면 관리자의 화면에 즉시 나타나고, 관리자가 '입금 완료'를 누르면 모든 직원의 화면에 실시간으로 노란색으로 불이 들어오게 됩니다.

이를 위해 구글에서 제공하는 무료 실시간 데이터베이스 서비스인 **Firebase (Firestore)**를 연동합니다.

## User Review Required
> [!IMPORTANT]
> - 이 기능을 구현하려면 고객님 명의의 구글 계정으로 **Firebase 데이터베이스를 새로 생성**하는 작업이 필요합니다. 
> - 무료로 제공되며, 생성 과정 자체는 제가 순서대로 안내해 드리면 3분 이내에 끝납니다. 
> - 데이터베이스가 중앙에 생성되면 Vercel에 올려둔 우리 웹사이트의 코드를 이 DB와 연결하도록 업데이트해야 합니다.

## Proposed Changes

### 1. 웹(프론트엔드) 코드 변경 (`app.js`, `index.html`)
- [MODIFY] `index.html`
  - Firebase SDK (웹 연결 라이브러리) 스크립트를 추가합니다.
- [MODIFY] `app.js`
  - 기존의 "내 브라우저에만 저장(tableData)"되던 로직을 지우고, 데이터를 "Firebase 클라우드 서버로 전송"하는 로직으로 변경합니다.
  - 데이터베이스의 정보가 변할 때마다 화면을 자동으로 새로고침 없이 즉각 그려주는 실시간 스트리밍(Live Sync) 기능을 탑재합니다.

### 2. 백엔드 유지 (`api/process-receipt.js`)
- 영수증 데이터를 추출하는 구글 Gemini API 통신부는 이전과 완벽하게 동일하게 유지하여 안전하게 API Key를 보호합니다.

## Verification Plan
### Manual Verification
1. 고객님께서 Firebase DB를 생성하고 설정값을 제게 전달해주십니다.
2. 제가 코드를 적용하면, GitHub를 통해 Vercel에 자동으로 최신 코드가 배포됩니다.
3. 배포된 웹사이트 주소를 컴퓨터와 스마트폰 2대로 동시에 열어놓습니다.
4. 스마트폰에서 사진을 올려 데이터가 추가되면, 즉시 컴퓨터 화면의 표에도 나타나는지 확인합니다.
5. 컴퓨터에서 '입금 완료'에 체크하면, 스마트폰 화면의 줄도 즉시 노란색으로 변하는지 확인합니다.
